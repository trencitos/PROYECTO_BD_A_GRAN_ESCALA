=========================================
Technical/Code Decisions
=========================================
1. Environment Variables: Created a `.env` file to store absolute paths and database credentials. This ensures no hardcoded paths or secrets exist in the source code, promoting security and portability.
2. Directory Structure: Resolved the duplication of `dbt_project` directories by consolidating everything into the root `dbt_project/` folder and removing `super_market_dbt`. This provides a cleaner structure and prevents execution errors.
3. Airflow Refactoring: The DAG is being refactored to use the `python-dotenv` library to read paths dynamically. Replaced inline comments with Python docstrings and added strict type hints to conform to the project requirements.
4. Beam Parallel Ingestion: A custom threaded reader or threaded processing mechanism will be introduced in Apache Beam to meet the constraint of using `threading` and `locks`, despite Beam having its own parallelization mechanisms. This satisfies the strict requirement to implement manual parallelized reading for CSVs.
5. Beam Idempotency: Implemented logic to ensure running the pipeline multiple times doesn't corrupt state.
6. Documentation and Language: Forced all code elements, variables, and documentation (except final LaTeX text) to English as requested.

=========================================
Data Engineering/Logic Decisions
=========================================
1. Audit Columns: Added `created_at` and `updated_at` timestamps in the Beam pipeline output metadata to improve data tracking and idempotency handling in the Gold layer.
2. Data Enhancements: Created a unified `is_active` boolean field to indicate non-refunded, fully completed sales, streamlining analytics queries downstream.
